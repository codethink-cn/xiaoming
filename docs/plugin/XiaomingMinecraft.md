### XiaomingMinecraft
# 小明 MC-QQ 互通插件
本插件是运行在 `Bukkit`、`Spigot`、`Paper`、`Bungee（即将发布）` 等端的 `Minecraft` 服务器 - `QQ` 互通插件。目前由椽子和 `ed-3` 开发。

项目基于小明 `QQ` 机器人框架 [xiaoming-bot](https://github.com/Chuanwise/xiaoming-bot) 开发，这是一款基于 `Mirai` 的插件化、便于上手、简单小巧的通用 QQ 机器人框架。具有非常简单的上下文交互接口。

除了支持最基础的远程执行指令、绑定查询、互通聊天外，本插件还支持跨区域 @、申请执行指令等功能。

**小明及相关插件的技术交流 / 用户 QQ 群**：[`1028959718`](https://jq.qq.com/?_wv=1027&k=sjBXo6xh)

## 使用方式
### 启动机器人的 QQ
请参照[这里](http://chuanwise.cn:10074/#/manual?id=%e5%87%86%e5%a4%87%e5%b7%a5%e4%bd%9c)的说明。

### 让小明连接 MC 服务器
1. 请在**小明根目录**下的 `plugins` 文件夹中放入 `XiaomingMinecraft-Server-xxx.jar`，然后重新启动小明。

1. 在你的 **`MC` 服务器** 的 `plugins` 目录下放入 `XiaomingMinecraft-Bukkit-xxx.jar`，然后重新启动 `MC` 服务器加载这个插件。

1. 小明和服务器只会连接它们认识的 `QQ` 和服务器，一开始小明不认识任何服务器，所以他会拒绝服务器的连接。服务器后台会给出连接失败的提示。

1. 添加机器人为好友后，私聊发送「迎接新服务器」允许陌生服务器接入。随后在 MC 服务器控制台输入 `/xm connect`。你将在 QQ 私聊收到提醒「有新的服务器接入，但小明还不敢确定这是你的服务器。赶快把这个服务器控制台上显示的验证码发过来吧」。按照说明查看 MC 服务器控制台查看验证码，将验证码私聊发送给小明即可开始初次配置。

1. 如果执行小明指令 `在线服务器` 时看到了自己的服务器，那么恭喜你，你的 `QQ` 从此和服务器互通了！

1. 可以执行小明指令 `指令格式` 查看相关小明指令的格式。

### 配置互通频道
小明的互通是通过「频道」确立的。服务器往群聊发送消息的频道，是「群聊频道」。群聊往服务器发送消息的频道，是「服务器频道」，二者相互独立。

这两种频道都关联若干服务器、群聊和世界。这涉及到「标记」，接下来简要解释标记的含义。标记相当于 #tag。在社交平台上，我们可以通过给一条动态打一些 #tag，通过搜索 #tag，可以找到带该 #tag 的所有动态。小明的标记同理。我们可以给群聊、服务器、服务器中的世界打标记，再把标记关联到频道，就可以让频道和若干服务器、群聊、世界关联起来。

值得一提，所有的东西都带 `recorded` 标记。除此之外，群聊会带群号标记，服务器会带服务器名标记，服务器里的世界会带世界名标记。这些默认的标记是「原生标记」，不可以被删除。在此基础上你可以增删自己的标记。

例如，某个服务器频道要求如下：
1. 只能在 `111` `222` 这两个群聊上往频道上发言；
1. 这两个群聊中的所有消息都需要被转发到 `aaa` 这个服务器中；
1. 消息需要转发到服务器的 `AAA` `BBB` 这两个世界中。

我们应该执行小明指令 `标记群  111  xxx`、`标记群  222  xxx` 给这两个群添加 `xxx` 标记。其中 `xxx` 可自定，只需要保证两边一致。

随后在服务器中执行 `/xm world AAA tag + yyy` 和 `/xm world BBB tag + yyy` 给这两个世界添加 `yyy` 标记。其中 `yyy` 也可自定，只需要保证两边一致。

随后执行小明指令 `添加服务器频道  <服务器频道名>`，例如 `添加服务器频道  <少儿频道>`，随后在小明询问群聊标记时发送 `xxx`，询问服务器标记时发送 `aaa`，询问世界标记时发送 `yyy` 即可。

其中因为服务器有一个默认标记是服务器名，所以无需给服务器打标记，`aaa` 就是服务器的标记。

添加群聊频道同理，再次不赘述。

## 常见问题
**如果你发现你的配置文件中没有出现下面答案中提及的相关项目，请使用群文件中最新的** `xiaoming-host`、`XiaomingMinecraft-Bukkit` 和 `XiaomingMinecraft-Server`。

### 启动
#### 无法打开 HOST，闪退怎么办？
**请严格按顺序检查下列项目：**

1. 设备上是否具有 `Java` 环境。打开控制台，输入 `java -version`，如果能看到 `Java` 版本，说明设备上具有 `Java` 环境，否则请下载并安装 `JVM` 或 `JRE`。版本要求 `Java 8` （也叫 `Java 1.8`）以上。
1. 启动脚本是否语法正确。`-jar` `-Dmirai.slider.captcha.supported` 和 `xiaoming-host-xxxx.jar` 之间都必须带有至少一个空格。
1. 检查 `launcher/launcher.json` 是否正确。下面是一个范例，**`password` 和 `protocol` 必须用英文双引号 `"` 包围**。
```json
{
    "account": {
        "qq": 123456789,
        "md5": null,
        "password": "password"
    },
    "protocol": "ANDROID_PHONE",
    "autoReconnectOnForceOffline": false
}
```

#### 显示「无法完成滑块验证」怎么办？
请将 `launcher/launcher.json` 中的登陆方式 `protocol` 改为手机登录：`ANDROID_PHONE`，再为启动脚本添加启动参数 `Dmirai.slider.captcha.supported`，也就是把机器人的启动脚本改成类似这样：
```bash
java -Dmirai.slider.captcha.supported -jar xiaoming-host-xxxx.jar
```
之后重新启动该脚本。在群内，或[这里](https://github.com/mzdluo123/TxCaptchaHelper)下载滑块验证助手，将启动机器人时显示的弹框内容复制到滑块验证助手中，再将获得的一串文字复制回弹框下方的白色输入框（**这行白色的框不是装饰**）后关闭弹框即可。

#### 显示「设备异常，禁止登陆」怎么办？
将 `xiaoming-host-xxx` 复制到自己的设备上登录小明后，找到 `launcher/device.json`。将该文件替换服务器那边的 `launcher/device.json`，随后重新启动小明。

### 连接
#### 为什么机器人不断告诉我「该服务器不在线」？
请使用新版小明。

#### 为什么机器人和小明连不上或总是掉线？
请确保小明所有组件的版本均为最新后，重启小明后重启 `MC` 服务器。

#### 为什么小明会拒绝服务器的连接？
小明只会连接它认识的服务器。请私聊小明 `迎接新服务器`，随后在 `MC` 服务器上使用 `/xm connect` 再次连接。

#### 为什么服务器会拒绝连接小明？
请使用新版小明。

### 配置
#### 如何让服务器和群内的消息无条件转发
在创建频道时，小明询问「什么开头的消息需要转发」时回答「转发所有」。

#### 如何在服务器显示消息转发结果？
`服务器根目录/plugins/xiaoming/configuration.yml` 中的 `showForwardResult` 项设置为 `true`。

#### 为什么小明不理我？
请确保在和机器人的私聊或群聊中发送指令。临时会话执行指令存在问题。

如果是私聊，请后台查看是否因为「登陆设备异常」或其他原因被强制下线。如果是群聊，请确保本群已经启动了小明（发送「本群启动小明」）。如果仍然没回复，请参考下一个问题。

#### 为什么我的机器人群聊不响应，私聊响应？
群聊和私聊各发一句 `call`。如果群聊不响应但是私聊响应，则机器人账号可能被腾讯风控，过一两小时到一两天自动会好。

#### 什么是默认目标服务器？
小明支持同时连接多个服务器，所以和服务器相关的操作，例如执行指令或查询在线人数，都需要指定一个目标。默认目标服务器就是所有人默认操作的对象。使用 `设置默认目标服务器 <服务器名>` 来修改设置。
